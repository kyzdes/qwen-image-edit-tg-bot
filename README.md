# Qwen Image Edit — Telegram-бот

Приватный Telegram-бот для редактирования фотографий через Hugging Face Space
[`productowner/Qwen-Image-Edit-2509-LoRAs-Fast-v2`](https://huggingface.co/spaces/productowner/Qwen-Image-Edit-2509-LoRAs-Fast-v2).
Отправь фото, выбери фичу (LoRA-адаптер), при желании напиши свой промпт, подстрой
параметры генерации — и получи результат прямо в чате.

## Что умеет

Бот работает по простому сценарию:

1. Присылаешь боту фото (или картинку документом).
2. Выбираешь фичу из меню кнопок.
3. По желанию набираешь свой текстовый промпт (на английском — так модель работает лучше).
4. При необходимости открываешь «⚙️ Настройки» и меняешь шаги, guidance, seed.
5. Жмёшь «✨ Сгенерировать» и получаешь обработанное изображение.

Результат можно «🔄 Повторить» (с новым случайным seed), запросить «🆕 Новое фото»
или вернуться в «📋 Меню фич».

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни значения:

| Переменная | Обязательна | По умолчанию | Описание |
| --- | --- | --- | --- |
| `BOT_TOKEN` | да | — | Токен бота из [@BotFather](https://t.me/BotFather) |
| `ALLOWED_USER_IDS` | нет | пусто | ID разрешённых пользователей через запятую/пробел. Пусто = доступ для всех |
| `SPACE_ID` | нет | `productowner/Qwen-Image-Edit-2509-LoRAs-Fast-v2` | ID Hugging Face Space |
| `HF_TOKEN` | нет | — | Токен Hugging Face (для публичного Space не нужен) |
| `REQUEST_TIMEOUT` | нет | `300` | Таймаут запроса к Space в секундах |

> Внимание: если `ALLOWED_USER_IDS` пуст, бот отвечает всем — для приватного
> использования обязательно укажи свой Telegram ID. Узнать его можно у бота
> [@userinfobot](https://t.me/userinfobot).

## Создать бота в @BotFather

1. Открой [@BotFather](https://t.me/BotFather) в Telegram.
2. Отправь `/newbot`, задай имя и username бота.
3. Скопируй выданный токен в переменную `BOT_TOKEN`.

## Запуск локально

```bash
# 1. Виртуальное окружение
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 2. Зависимости
pip install -r requirements.txt

# 3. Переменные окружения
cp .env.example .env             # затем впиши свой BOT_TOKEN и ALLOWED_USER_IDS

# 4. Экспортируй переменные и запусти
export $(grep -v '^#' .env | xargs)   # или задай переменные вручную
python bot.py
```

Бот работает по long polling — внешний домен и вебхук не нужны.

## Запуск через Docker

```bash
# Сборка образа
docker build -t qwen-image-edit-bot .

# Запуск с передачей .env
docker run --rm --env-file .env qwen-image-edit-bot
```

## Деплой через Dokploy

Бот использует long polling, поэтому домен и проброс портов не требуются.

1. Создай в Dokploy приложение из этого Git-репозитория (build type — Dockerfile).
2. В разделе Environment добавь переменные из `.env.example`
   (как минимум `BOT_TOKEN`, при необходимости `ALLOWED_USER_IDS`).
3. Запусти деплой. Dokploy соберёт образ по `Dockerfile` и запустит `python bot.py`.
4. Домен настраивать НЕ нужно — бот сам подключается к Telegram по polling.

При пуше в отслеживаемую ветку Dokploy пересоберёт и перезапустит контейнер автоматически.

## Заметки

- Лимит ZeroGPU на Space сбрасывается примерно через 24 часа после первого
  использования. Если упёрся в квоту, бот так и напишет.
- Промпты лучше писать на английском — модель обучена на английских инструкциях.
